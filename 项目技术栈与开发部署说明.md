# 南托岭·邻里闲置 项目技术栈与开发部署说明

## 1. 项目简介

这是一个面向社区邻里的二手交易网站，主要服务宝妈和年轻家庭，强调手机端浏览、低门槛发布、当面交易。

当前项目包含以下核心能力：

- 首页浏览闲置物品
- 按年龄段筛选
- 发布闲置并上传图片
- 查看详情、复制联系方式、标记已出
- 查看“我的发布”
- 微信群引流入口

目标用户和场景特征：

- 目标用户以宝妈、带娃老人、幼儿家长为主
- 核心品类集中在婴童用品、玩具绘本、衣服鞋帽、出行工具
- 主要使用路径是“手机浏览 -> 查看邻居在卖什么 -> 拍照发布 -> 线下面交”

## 2. 技术栈总结

### 前端框架

- `Next.js 14`
- `React 18`
- `TypeScript 5`

项目使用 Next.js App Router 目录结构，页面位于 `src/app` 下。

### UI 与样式

- `Tailwind CSS 3`
- 全局样式文件：`src/app/globals.css`
- Tailwind 配置文件：`tailwind.config.ts`

样式特点：

- 移动端优先
- 自定义品牌色
- 卡片式 UI
- 适配安全区域底部留白

### 数据与后端能力

- `Supabase`
- `@supabase/supabase-js`

当前项目没有独立业务后端，前端直接通过 Supabase SDK 访问：

- `items` 数据表
- `item-images` 对象存储 bucket

`src/app/api/items/route.ts` 当前只是占位接口，真实数据读写仍走 Supabase。

### 图片能力

- 图片上传到 Supabase Storage
- Next.js `Image` 组件已允许加载 `**.supabase.co` 远程图片

### 本地状态存储

- 浏览器 `localStorage`

当前设备发布过的物品 ID 会保存在本地，用于“我的发布”页面识别。

## 4. 环境变量

本项目依赖 `.env.local`。

示例：

```bash
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
NEXT_PUBLIC_DEFAULT_COMMUNITY=南托岭社区
```

说明：

- `NEXT_PUBLIC_SUPABASE_URL`：Supabase 项目地址
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`：Supabase 匿名访问公钥
- `NEXT_PUBLIC_DEFAULT_COMMUNITY`：默认展示的小区名称

注意：

- 这几个变量都带 `NEXT_PUBLIC_` 前缀，表示会暴露到前端
- 因此不要把服务端私钥写入前端环境变量

## 5. Supabase 数据结构与初始化

### 5.1 `items` 表

`items` 是项目的核心数据表，用于保存物品信息、位置、联系方式、状态和统计字段。

推荐在 Supabase SQL Editor 中执行以下 SQL：

```sql
CREATE TABLE IF NOT EXISTS items (
  id          BIGSERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  age_group   TEXT NOT NULL DEFAULT '3-6岁',
  category    TEXT NOT NULL DEFAULT '其他',
  price       DECIMAL(10,2),
  description TEXT DEFAULT '',
  images      TEXT[] NOT NULL DEFAULT '{}',
  community   TEXT NOT NULL DEFAULT '南托岭社区',
  building    TEXT DEFAULT '',
  contact     TEXT NOT NULL,
  status      TEXT NOT NULL DEFAULT 'active',
  view_count  INT NOT NULL DEFAULT 0,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_items_age_group ON items(age_group);
CREATE INDEX IF NOT EXISTS idx_items_category  ON items(category);
CREATE INDEX IF NOT EXISTS idx_items_community ON items(community);
CREATE INDEX IF NOT EXISTS idx_items_status    ON items(status);
CREATE INDEX IF NOT EXISTS idx_items_created   ON items(created_at DESC);

CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

DROP TRIGGER IF EXISTS trigger_items_updated_at ON items;
CREATE TRIGGER trigger_items_updated_at
  BEFORE UPDATE ON items
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

字段说明：

- `title`：物品标题
- `age_group`：适用年龄段
- `category`：物品分类
- `price`：价格，可为空；空值可视为免费送
- `description`：文字描述
- `images`：图片 URL 数组
- `community` / `building`：小区与楼栋信息
- `contact`：联系电话或微信号
- `status`：当前状态，使用 `active` / `sold`
- `view_count`：浏览次数
- `created_at` / `updated_at`：创建和更新时间

### 5.2 Storage

需要创建一个公开的 Storage bucket：

- bucket 名称：`item-images`
- 权限：`public`

用途：

- 发布页上传物品图片
- 列表页和详情页直接使用公开 URL 渲染图片

### 5.3 初始化步骤

1. 创建 Supabase 项目
2. 执行本节中的建表 SQL
3. 创建 `item-images` public bucket
4. 复制项目 URL 和 anon key 到 `.env.local`
5. 按实际需求检查匿名访问策略

## 7. 当前业务流程说明

### 首页

- 页面加载后直接从 Supabase 查询 `items`
- 只展示 `status = active` 的数据
- 按发布时间倒序排列
- 支持按年龄段筛选

### 发布页

- 用户选择图片
- 图片上传到 `item-images` bucket
- 获取公开图片地址
- 将表单数据写入 `items`

### 详情页

- 根据 `id` 查询单个物品
- 自动增加浏览次数
- 支持复制联系方式
- 当前设备发布者可标记已出或删除

### 我的发布

- 从 `localStorage` 读取当前设备已发布的物品 ID
- 再去 Supabase 查询对应记录

## 8. 关键设计决策

- 不要求用户注册登录，优先降低发布和成交门槛
- 不集成在线支付，只负责信息撮合和线下面交
- 不内置站内聊天，默认通过电话或微信直接联系
- 采用手机优先页面结构和底部导航，方便微信群传播
- 以年龄段筛选优先于复杂类目树，更符合家长直觉

## 9. 生产部署方案

结合当前项目形态，推荐两种方式。

### 方案 A：快速上线

- 前端部署到 Node.js 服务器
- 数据仍使用 Supabase
- 适合先跑通 MVP

### 方案 B：面向国内用户正式上线

- Next.js 部署到中国大陆云服务器
- 域名完成实名和 ICP 备案
- 图片走国内 CDN 或后续迁移到国内对象存储

如果目标是国内用户稳定访问，优先推荐方案 B。

补充说明：

- 这个项目早期曾按 `Vercel + Supabase` 的轻量方案设计，适合 MVP 验证
- 当前如果要服务中国大陆用户长期访问，仍然以国内服务器部署为主

## 11. Nginx 反向代理示例

```nginx
server {
    listen 80;
    server_name nantuoling.cn www.nantuoling.cn;
    return 301 https://www.nantuoling.cn$request_uri;
}

server {
    listen 443 ssl http2;
    server_name www.nantuoling.cn;

    ssl_certificate /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

建议：

- 根域名跳转到 `www`
- 全站强制 HTTPS
- 静态资源后续可接 CDN

## 12. 域名与备案建议
