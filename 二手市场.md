# 南托岭·邻里闲置 — 完整项目文档

## 一、项目概述

为长沙「南托岭社区」及周边小区做一个**社区闲置物品交易网页**。

**目标用户**：宝妈（主力）、带娃老人、即将上学的小孩家长
**核心场景**：婴童用品（推车/玩具/绘本/衣服）在邻居之间流转
**核心体验**：手机浏览器打开 → 看到邻居在卖什么 → 拍照发布 → 一键拨号联系

## 二、技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 框架 | Next.js 14 (App Router) + TypeScript | 全栈，SSR + 静态 |
| 样式 | Tailwind CSS | 手机优先响应式 |
| 数据库 | Supabase (PostgreSQL) | 免费 500MB，REST API |
| 图片 | Supabase Storage | 免费 1GB，Public bucket |
| 部署 | Vercel | 免费，推 GitHub 自动部署 |

## 三、数据库设计

### 3.1 Supabase 建表 SQL

在 Supabase SQL Editor 中执行：

```sql
CREATE TABLE IF NOT EXISTS items (
  id          BIGSERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  age_group   TEXT NOT NULL DEFAULT '3-6岁',
  category    TEXT NOT NULL DEFAULT '其他',
  price       DECIMAL(10,2),
  description TEXT DEFAULT '',
  images      TEXT[] NOT NULL DEFAULT '{}',
  community   TEXT NOT NULL DEFAULT '南托岭社区',
  building    TEXT DEFAULT '',
  contact     TEXT NOT NULL,
  status      TEXT NOT NULL DEFAULT 'active',
  view_count  INT NOT NULL DEFAULT 0,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_items_age_group ON items(age_group);
CREATE INDEX IF NOT EXISTS idx_items_category  ON items(category);
CREATE INDEX IF NOT EXISTS idx_items_community ON items(community);
CREATE INDEX IF NOT EXISTS idx_items_status    ON items(status);
CREATE INDEX IF NOT EXISTS idx_items_created   ON items(created_at DESC);

CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN NEW.updated_at = NOW(); RETURN NEW; END;
$$ LANGUAGE plpgsql;

DROP TRIGGER IF EXISTS trigger_items_updated_at ON items;
CREATE TRIGGER trigger_items_updated_at
  BEFORE UPDATE ON items
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

### 6.13 src/app/post/page.tsx — 发布页

表单字段（由上到下）：
1. 图片上传（3列网格，最多9张，点+选图，支持删除）
2. 物品名称（文本输入框，必填）
3. 适用年龄（下拉选择）+ 价格（数字输入，空=免费送）
4. 分类（按钮组：推车座椅/玩具绘本/衣服鞋帽/喂养用品/学习用品/出行工具/其他）
5. 所在小区（下拉选择）+ 楼栋（文本输入，选填）
6. 描述（文本域，选填）
7. 联系方式（文本输入，必填，提示留微信号）
8. 提交按钮（绿色大按钮"📢 发布闲置"）

图片上传到 Supabase Storage `item-images` bucket，拿到 publicUrl 存入 items 表。
发布成功后 `router.push("/")` 回到首页。

## 七、关键设计决策

1. **不需要用户注册登录**：匿名发布，联系方式写手机号/微信号，降低门槛
2. **不需要在线支付**：只做信息撮合，邻居面交现金
3. **不需要聊天IM**：直接拨打电话或复制微信号
4. **手机优先**：max-w-lg 容器，底部固定导航，安全区适配
5. **年龄段分类而非品类分类**：宝妈一眼就懂"0-2岁"是什么，不需要思考"这属于哪个品类"
